# Barcode Quest – PRD

## Problem
A mobile-first barcode-scanning RPG: scan real product barcodes to acquire
fantasy gear (armor / weapons / accessories / consumables / upgrade scrolls).
Items scale to character level; upgrades let players push gear up to (but
not past) their character level. Players buy / sell / trade items in an
in-game Trading House with min/max price guardrails. Items can also be
destroyed for a small gold refund.

## Users
Casual mobile players who like daily collection loops, anime/fantasy
aesthetics, and barcode-driven serendipity.

## Tech
- Frontend: Expo SDK 54, expo-router, expo-camera, react-native components
- Backend: FastAPI + MongoDB (motor), JWT auth (bcrypt + pyjwt)
- AI flavor: Claude Haiku 4.5 via emergentintegrations (Emergent LLM key)

## Core flows (v1, shipped)
1. Register / login (JWT in expo-secure-store; web shim via AsyncStorage)
2. Hero tab: avatar ring, level + XP bar, HP/MP, ATK/DEF/MAG totals, equipped slot grid, gold
3. Scan tab: live camera scanner + fallback manual entry → backend `/api/scan`
   - Slot derived from barcode digits (deterministic)
   - Rarity rolled with bias from character level
   - Stats scaled by rarity + level
   - Name + lore generated by Claude Haiku 4.5
4. Inventory tab: grid filtered by slot; tap → item detail bottom-sheet
5. Item detail: equip / unequip / use (consumable) / destroy / list / upgrade
   - Upgrade caps at character level
   - Destroying refunds gold (level × 2)
6. Trading House: browse, my listings, post with min/max price band, buy, cancel
   - Buy fails if gold insufficient
   - Cannot buy own listing

## Business hook
Future: cosmetic-only "Hero Pass" subscription unlocking premium avatar frames,
extra Trading House slots, and 2x daily scan XP. Free players keep all
gameplay; monetization is purely cosmetic / convenience.

## Out of scope (v1)
- Push notifications
- PvP combat / dungeons
- Cross-device synchronization beyond JWT
- Multiple character classes
